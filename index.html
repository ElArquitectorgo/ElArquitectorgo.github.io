Introducción

Nuestro objetivo en esta práctica va a ser encontrar un modelo que sea capaz de predecir el evento PCR para poder desplegarlo en una clínica, para ello usaremos varias técnicas ampliamente conocidas en el ámbito de la cienca de datos que iremos detallando más adelante.

Como recordatorio de la práctica anterior, obtuvimos el siguiente modelo como mejor predictor: Estadio + Grado + Fenotipo, pero al intentar usar ese modelo para predecir el evento PCR de las muestras de nuestro dataset nos encontramos con un problema.

En el resumen de las predicciones vemos que el máximo obtenido es 0.54 y que la media es 0.15, lo cual quiere decir que no está prediciendo prácticamente ningún PCR positivo. Para ser más rigurosos vamos a crear una función que dado un dataset y unas predicciones nos devuelva las métricas de precisión, especificidad y sensibilidad, estas últimas dos medidas nos indican respectivamente la proporción de verdaderos negativos clasificados correctamente, y la proporción de casos positivos clasificados correctamente. Usaremos esas medidas para estudiar la capacidad predictiva de nuestro modelo a la hora de discriminar los casos positivos de los negativos.

precisión <- function(real, predicción, print_matrix = FALSE) {
  TP = length(which(predicción >= 0.5 & real$PCR == 1))
  TN = length(which(predicción <= 0.5 & real$PCR == 0))
  acc = (TP + TN) / length(predicción)
  
  FP = length(which(predicción >= 0.5 & real$PCR == 0))
  FN = length(which(predicción <= 0.5 & real$PCR == 1))
  
  especificidad = TN / (TN + FP)
  sensibilidad = TP / (TP + FN)
  
  v = cbind(c(TN, FN),c(FP, TP))
  colnames(v) <- c("F","T")
  rownames(v) <- c("F","T")
  if (print_matrix) print(v)
  
  return(c(acc, especificidad, sensibilidad))
}

Un ACC del 80% parecería indicar que nuestro modelo, tal y como calculamos en la práctica anterior, era un buen modelo, pero vemos que tiene una sensibilidad nula, y esa precisión del 80% coincide justo con la proporción de PCRs negativos que hay en nuestro dataset, por tanto, que el modelo prediga casi todos los PCR como 0 significa que se está ajustando para responder correctamente a la mayoría de datos. La clase está tan desbalanceada que el algoritmo está aprendiendo la clase más frecuente.

Esto nos lleva directamente a sacar dos conclusiones, la primera es que debemos buscar una manera alternativa de encontrar un modelo que no se base únicamente en la precisión, y la segunda es que vemos que el AIC no nos sirve para obtener el mejor modelo predictivo.

Nuestro objetivo ahora será buscar el modelo con mejor ACC, con sensibilidad y especificidad distinta de 0. Para evitar los efectos de tener clases desbalanceadas añadiremos una métrica que combina la sensibilidad y la precisión, el F1-score.

Brute Force

Lo primero que se nos viene a la cabeza para encontrar el mejor modelo, es probar todos los modelos, esto tiene el evidente inconveniente del coste computacional que conlleva, lo haremos de todas formas ya que en este caso es asumible y lo podremos utilizar para comparar resultados posteriores.

all.combinations <- unlist(lapply(1:ncol(datos[,-c(1,10)]), function(i) combn(colnames(datos[,-c(1,10)]), i, simplify = FALSE)), recursive=FALSE)
test.all.combinations <- function() {
  minAIC = 99999
  maxACC = 0
  acc = 0
  F1score = 0
  bestModel = all.combinations[1]
  for (i in 1:length(all.combinations)) {
    str = paste(unlist(all.combinations[i]), collapse="+")
    str = paste("PCR ~", str)

    lr <- glm(as.formula(str), data = datos, family = binomial("logit"))
    result = AIC(lr)
    
    if (result < minAIC) minAIC = result
    
    predictions <- predict(lr, newdata=datos, type="response")
    results = precisión(datos, predictions)
    
    if (results[1] > maxACC) maxACC = results[1]
    
    score = 2 * results[1] * results[3] / (results[1] + results[3])
    
    if (score > F1score) {
      acc = results[1]
      F1score = score
      bestModel = all.combinations[i]
    }
  }
  print(paste("ACC =", acc))
  print(paste("F1-score =", F1score))
  print(bestModel)
}

Hold Out

Para evitar en la medida de lo posible que un modelo quede sobreajustado (ya que estamos usando todo el dataset como entrenamiento y test), vamos a implementar el método hold out, que dado un dataset, dividirá éste en dos, uno para entrenamiento y otro para test. El tamaño de cada subconjunto se lo pasaremos como parámetro.

Nótese que en la función estamos eliminando información del dataset, ya que al tener pocas muestras, puede ocurrir que haya categorías que aparezcan en los datos de entrenamiento pero no en los de test, o viceversa, y por tanto se lanzarían excepciones.

hold.out <- function(datos, train) {
  levels(datos$REst)[levels(datos$REst) == "I"] <- NA
  levels(datos$RPro)[levels(datos$RPro) == "I"] <- NA
  levels(datos$Estadio)[levels(datos$Estadio) == "T0"] <- NA
  levels(datos$Estadio)[levels(datos$Estadio) == "T1"] <- NA
  levels(datos$Grado)[levels(datos$Grado) == "1"] <- NA
  indexs = createDataPartition(datos$PCR, p=train, list=FALSE, times=1)

  training.set <- datos[indexs,]
  test.set <- datos[-indexs,]
  
  return (list(training.set, test.set))
}

Repeated Hold Out

La implementación de repeated Hold Out va a consistir en realizar n veces un hold out dentro de un bucle. Vamos a eliminar las variables Muestra y Her2 del dataset ya que sabemos no nos aportan información, así como los NA’s, que nos van a dar problemas a la hora de dividir el conjunto de datos.

datos2 <- na.omit(datos)
datos2$Muestra <- NULL
datos2$Her2 <- NULL

Y vamos a crear una función que nos devuelva el F1-score y nos dibuje la curva ROC en caso de quererlo.

get.results <- function(training, test, formula, plot_roc = FALSE) {
  lr <- glm(as.formula(formula), data = training, family = binomial("logit"))
  predictions <- predict(lr, newdata=test, type="response")
  results = precisión(test, predictions)
  f1score = 2 * results[1] * results[3] / (results[1] + results[3])
  
  if (plot_roc) {
    obj.roc <- roc(test$PCR, predictions, smooth=FALSE, auc=TRUE)
    plot(obj.roc)
    print(auc(obj.roc))
  }

  return (c(results, f1score))
}

Finalmente definimos una nueva función donde vamos a evaluar todos los modelos posibles, realizándole a cada modelo un repeated hold out de n iteraciones siguiendo un ratio entrenamiento/test dado. Se escogerá el mejor modelo basándonos en la media del f1-score de cada uno.

test.all.combinations.hold <- function(n, ratio) {
  combinations <- unlist(lapply(1:ncol(datos2[,-c(0,8)]), function(i) combn(colnames(datos2[,-c(0,8)]), i, simplify = FALSE)), recursive=FALSE)
  acc = 0
  best_f1score = 0
  best_model = combinations[1]
  # Vamos a ahorrarnos unas cuantas iteraciones si obviamos las primeras combinaciones que sabemos no van a ser las mejores.
  for (i in 28:length(combinations)) {
    str = paste(unlist(combinations[i]), collapse="+")
    str = paste("PCR ~", str)
    acc_train_test = 0
    f1score_train_test = 0
    for (j in 1:n) {
      res <- hold.out(datos2, ratio)
      training.set = data.frame(res[1])
      test.set = data.frame(res[2])
      acc_train_val = 0
      f1score_train_val = 0
      
      results = get.results(training.set, test.set, str)
      acc_train_test = acc_train_test + results[1]
      f1score_train_test = f1score_train_test + results[4]
    }
    acc_train_test = acc_train_test / n
    f1score_train_test = f1score_train_test / n
    
    if (f1score_train_test > best_f1score) {
      acc = acc_train_test
      best_f1score = f1score_train_test
      best_model = combinations[i]
    }
  }
  print(acc)
  print(best_f1score)
  print(best_model)
}

get.model.distribution <- function(formula, n, ratio) {
  acc <- c()
  f1score <- c()
  for (i in 1:n) {
    res <- hold.out(datos2, ratio)
    training.set = data.frame(res[1])
    test.set = data.frame(res[2])
    results = get.results(training.set, test.set, formula)
    acc <- c(acc, results[1])
    f1score <- c(f1score, results[4])
  }
  return (f1score)
}

K-fold Cross Validation

Al realizar la implementación de la validación cruzada he podido eliminar menos datos.

k.cross.validation <- function(datos, formula, k) {
    acc_train_test = 0
    f1score_train_test = 0
    levels(datos$REst)[levels(datos$REst) == "I"] <- NA
    levels(datos$RPro)[levels(datos$RPro) == "I"] <- NA
    levels(datos$Estadio)[levels(datos$Estadio) == "T0"] <- NA
    
    #levels(datos$Estadio)[levels(datos$Estadio) == "T1"] <- NA
    #levels(datos$Grado)[levels(datos$Grado) == "1"] <- NA
    folded.data <- createFolds(datos$PCR, k)
    for (i in 1:length(folded.data)) {
        training.set <- data.frame(datos[-unlist(folded.data[][i]),])
        test.set <- data.frame(datos[unlist(folded.data[][i]),])
        results = get.results(training.set, test.set, formula)
        acc_train_test = acc_train_test + results[1]
        f1score_train_test = f1score_train_test + results[4]
    }
    acc_train_test = acc_train_test / k
    f1score_train_test = f1score_train_test / k
    
    return (c(acc_train_test, f1score_train_test))
}

Conclusiones

A lo largo de este trabajo hemos ido viendo distintas formas de hallar un modelo intentando evitar los efectos del sobreajuste. Finalmente hemos obtenido unos resultados de precisión y AUC aceptables, pero que a mi parecer, observando la matriz de confusión, siguen sin ser suficientemente buenos. Contamos con pocos datos, y además en algunas circunstancias tenemos que eliminar datos para evitar errores en la ejecución de los algoritmos, por tanto, sería sensato esperar a tener un dataset mayor, o buscar una manera de minimizar esa pérdida de información antes de lanzar este predictor al mercado.

También hay que tener en cuenta que en este proyecto ejecutar varias veces la misma función puede aportar resultados distintos, este problema nos sugiere buscar una forma más consistente de analizar los datos. Es por eso que veo los resultados obtenidos como aproximaciones que pueden ayudarme a sacar conclusiones, pero nunca para ser usados como único criterio a la hora de escoger el mejor modelo.

En el análisis bivariante vimos que las variables relacionadas con PCR eran REst, RPro, Grado y Fenotipo, nos parecía sospechoso que NodAfec no estuviese relacionada. Finalmente todos los modelos obtenidos en esta práctica sí han incluido esta variable, y algunos han omitido REst o RPro. La variable Estadio no estaba relacionada en principio con PCR y sin embargo casi todos los modelos la han incluido. Por algún motivo la variable Edad mejora el modelo pese a haber visto que tampoco tenía relación con PCR.

El modelo escogido finalmente ha sido Edad + REst + Estadio + NodAfec + Grado + Fenotipo, lo que quiere decir que hemos eliminado las variables Muestra, RPro y Her2. Como hemos comentado, necesitaríamos datos nuevos para poder obtener unas métricas más reales sobre la capacidad predictiva del modelo.

Consideraciones para el futuro: tener una parte de datos de validación, vamos a sobreestimar los resultados de otra forma. Buscar el mejor modelo en función de validación y no de test, y luego comprobar la capacidad predictiva del modelo con la parte de test. No lo hemos hecho para evitar complejizar demasiado el problema.
